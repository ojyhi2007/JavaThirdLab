## Задание 1.9 — Диапазон оценок (Student)

<details>
<summary>Подробнее об задании</summary>

### Класс Student

### Описание:
Создан класс **Student**, который описывает студента по имени и набору оценок.
Главная цель задания — гарантировать, что студенту можно задавать только оценки в диапазоне **от 2 до 5**.

### Логика решения:
- Имя студента хранится в поле типа `String`.
- Оценки хранятся в массиве `int`.
- При создании студента и при добавлении новой оценки выполняется проверка:
  если оценка меньше 2 или больше 5 — выбрасывается `IllegalArgumentException`.
- Метод `getGrades()` возвращает **копию массива**, чтобы оценки нельзя было изменить извне.
- Метод `toString()` выводит студента в формате: `Имя: [оценки]`.

### Пример:
- Создание студента **Вася** с оценками `3,4,5`.
- Попытка добавить оценку `6` приводит к ошибке.
- Попытка изменить массив, полученный из `getGrades()`, не меняет оценки внутри объекта.

</details>

## Задание 2.3 — Желтые страницы (PhoneDirectory)

<details>
<summary>Подробнее об задании</summary>

### Класс PhoneDirectory

### Описание:
Создан класс **PhoneDirectory**, который реализует простой телефонный справочник.
Справочник хранит пары **имя — телефон** и позволяет выполнять операции поиска и управления контактами.

### Логика решения:
- Для хранения данных используется структура `LinkedHashMap`.
- Каждому имени соответствует **один телефон**, и каждому телефону соответствует **одно имя**.
- Класс реализует основные операции работы со справочником:
  - добавление контакта
  - удаление контакта
  - получение телефона по имени
  - проверка существования имени
  - проверка существования телефона
  - получение количества контактов
  - получение всех имен
  - получение всех телефонов
  - получение всех пар "телефон — имя"
  - поиск имен по заданному префиксу
- Метод `toString()` выводит полный список контактов справочника.

### Пример:
Программа выполняет следующие действия:

Добавляет контакты:


89003337788 - Вася
89111111111 - Маша
89222222222 - Петя


Вывод справочника:


[89003337788 - Вася, 89111111111 - Маша, 89222222222 - Петя]


Далее программа:
- меняет телефон Васи
- удаляет Машу из справочника
- выполняет поиск телефона по имени
- выводит список всех контактов.
</details>

## Задание 3.5 — Трехмерная точка

<details>
<summary>Подробнее об задании</summary>

### Классы Point и Point3D

### Описание:
Создан класс **Point**, который описывает точку на плоскости с координатами **X** и **Y**.

Также создан класс **Point3D**, который является расширением обычной точки и добавляет третью координату **Z**.  
Таким образом получается точка в **трёхмерном пространстве**.

### Логика решения:
- Класс `Point` содержит координаты **X** и **Y**.
- Класс `Point3D` наследуется от класса `Point`.
- Наследование реализовано с помощью ключевого слова `extends`.
- Координаты **X** и **Y** берутся из родительского класса.
- Координата **Z** добавляется в классе `Point3D`.
- Конструктор `Point3D` принимает три координаты: X, Y и Z.
- Метод `toString()` переопределен для корректного вывода всех координат точки.

### Пример:
Программа создает обычную точку и трехмерную точку:
Point: {3;5}
Point3D: {3;5;7}

После изменения координаты Z:
Point3D: {3;5;10}
Таким образом демонстрируется использование **наследования** и расширение функциональности базового класса.

</details>


## Задание 4.4 — Геометрические фигуры
<details>
<summary>Подробнее об задании</summary>

### Интерфейс Figure

### Описание:
Создан интерфейс **Figure**, который описывает геометрическую фигуру.

Интерфейс содержит метод:   
getArea()

Этот метод должен возвращать площадь фигуры.

На основе интерфейса реализованы следующие фигуры:   
- Circle (круг)   
- Square (квадрат)   
- Rectangle (прямоугольник)   
- Triangle (треугольник)   

Каждая фигура хранит необходимые параметры (точки или размеры) и умеет вычислять свою площадь.

### Логика решения:   
Создан интерфейс `Figure`, который содержит метод вычисления площади.

Каждая фигура реализует этот интерфейс с помощью ключевого слова: implements

Для каждой фигуры используется собственная формула площади:   
- круг — π * r²
- квадрат — сторона²
- прямоугольник — ширина * высота
- треугольник — вычисление площади по координатам точек

Таким образом реализуется принцип **полиморфизма**:  
разные фигуры имеют один общий метод `getArea()`.

### Пример:
Программа создает несколько фигур:   
Circle
Square
Rectangle
Triangle

Для каждой фигуры вызывается метод вычисления площади:   
Circle area = ...
Square area = ...
Rectangle area = ...
Triangle area = ...

Все фигуры можно хранить в массиве типа: Figure[]
и вызывать у них метод `getArea()` независимо от конкретного типа фигуры.

</details>

##  Задание 5.3 — Общая площадь фигур
<details>
<summary>Подробнее об задании</summary>

### Описание:
Создан класс **FigureUtils**, который содержит метод для вычисления общей площади нескольких геометрических фигур.   
Этот метод позволяет передать несколько фигур и получить сумму их площадей.

### Логика решения:
В классе `FigureUtils` реализован статический метод:   
totalArea(Figure... figures)

Метод принимает переменное количество фигур типа `Figure`.   
Для каждой фигуры вызывается метод:   
getArea()

который возвращает площадь конкретной фигуры.   
Полученные значения суммируются и возвращаются как общий результат.   
Таким образом демонстрируется использование **полиморфизма**, поскольку метод работает с объектами разных типов, реализующих интерфейс `Figure`.

### Пример:   
Создаются несколько фигур:   
Circle
Square
Rectangle

После передачи их в метод: totalArea(figures) программа вычисляет и выводит: общую площадь всех фигур.      
Метод корректно работает независимо от конкретного типа фигуры.

</details>
