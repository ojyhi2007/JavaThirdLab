## Задание 1.9 — Диапазон оценок (Student)

<details>
<summary>Подробнее об задании</summary>

### Класс Student

Описание  
Создан класс `Student`, описывающий студента по имени и массиву оценок.

### Логика решения

Имя хранится в виде строки.  
Оценки хранятся в массиве целых чисел.   
Гарантируется, что студенту можно задать **только оценки в диапазоне от 2 до 5**.   
При попытке задать некорректную оценку выбрасывается исключение `IllegalArgumentException`.   
Метод `getGrades()` возвращает **копию массива**, что защищает внутренние данные объекта.   
Метод `toString()` выводит информацию в формате:   
Имя: [оценка1, оценка2, ...]

</details>

## Задание 2.3 — Желтые страницы (PhoneDirectory)

<details>
<summary>Подробнее об задании</summary>

### Класс PhoneDirectory

Описание  
Создан класс `PhoneDirectory`, представляющий телефонный справочник.

Справочник хранит пары:
имя → телефон

### Логика решения

Используется структура данных `LinkedHashMap`.

Реализованы следующие операции:

- добавление контакта  
- удаление контакта  
- получение телефона по имени  
- проверка существования имени или телефона  
- получение количества контактов  
- получение массива всех имен  
- получение массива всех телефонов  
- получение всех пар "телефон – имя"  

Также реализован поиск имен по заданному префиксу.

Метод `toString()` выводит список всех контактов.

</details>

## Задание 3.5 — Трехмерная точка

<details>
<summary>Подробнее об задании</summary>

### Классы Point и Point3D

Описание  
Создан класс `Point`, описывающий точку на плоскости с координатами **X** и **Y**.

Создан класс `Point3D`, который **наследуется от Point** и добавляет координату **Z**.

### Логика решения

Координаты хранятся в виде чисел типа `double`.

Класс `Point3D` расширяет функциональность базового класса `Point`.

Метод `toString()` выводит координаты точки:
{X;Y} или {X;Y;Z}

</details>
